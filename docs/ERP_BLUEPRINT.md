# مجموعة السلام لاستيراد المواد الغدائية

## التحليل المختصر
- النظام الحالي كان يغطي البيع والشراء والمخزون بشكل محدود.
- التوسعة الجديدة حولته إلى نواة ERP مبسطة تدعم الشركة الأم والفروع والمسوقين وشجرة الحسابات والمستخدمين والصلاحيات.
- تم الحفاظ على المعمارية الحالية القائمة على PHP server-rendered مع واجهات HTML/CSS/JS عربية حتى لا يتعطل المشروع القائم.

## الوحدات المفعلة
- لوحة تحكم تنفيذية.
- إدارة الفروع والمخازن.
- إدارة المسوقين وربطهم بالفروع.
- إدارة العملاء مع ربط الفرع والمسوق.
- إدارة الموردين مع ربط الفرع.
- المبيعات مع ربط الفرع والمخزن والمسوق.
- المشتريات مع ربط الفرع والمخزن ومصاريف الاستيراد.
- المصروفات مع ربط الحساب المالي.
- الحسابات: شجرة حسابات + قيود يومية + قيود تلقائية من البيع والشراء والمصروفات.
- المستخدمون والأدوار والصلاحيات والفروع المسموح بها.
- الإعدادات والهوية الرسمية للشركة.

## تصميم قاعدة البيانات
- تمت إضافة جداول: `companies`, `branches`, `warehouses`, `marketers`, `marketer_branches`, `user_permissions`, `user_branches`, `accounts`, `journal_entries`, `journal_entry_lines`, `account_opening_balances`, `cashboxes`, `banks`, `branch_transfers`, `branch_transfer_items`, `receipts`, `payments`, `commissions`, `notifications`, `tasks`.
- وتم توسيع الجداول الحالية لتصبح branch-aware و finance-aware مثل: `users`, `customers`, `suppliers`, `products`, `sales`, `purchases`, `expenses`, `stock_movements`, `product_batches`.

## ملاحظات تشغيل
- ملف `database/database.sql` هو سكربت توسعة migration على القاعدة الحالية وليس إسقاطاً كاملاً للبيانات.
- القيود المحاسبية التلقائية تعتمد على مفاتيح الإعدادات الافتراضية للحسابات.
- إدارة المستخدمين تتيح صلاحيات إضافية استثنائية فوق صلاحيات الدور.
